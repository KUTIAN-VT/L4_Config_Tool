# 03\-\-L4 配置工具软件使用手册



本文面向 L4 配置工具的日常使用人员，说明软件启动、设备连接、状态查看、链路配置、固件升级和配置维护等常用操作。界面截图以实际运行窗口为准，设备在线状态、版本号、MAC 地址和日志内容会随现场设备变化。

## 1\. 运行前准备

使用前请确认：

- 将软件发布包完整解压，目录中包含 `L4_Config_Tool_Web.exe`、`l4_daemon.exe`、`l4_ota_upgrade.exe` 以及相关 DLL。

- L4 设备已通过 USB/串口等方式连接到电脑，并保持稳定供电。

安装驱动:

1. 运行 解压后文件夹中的 `zadig.exe`。

2. 点击菜单栏 `Options`，选择 `List All Devices`。如果界面已经显示 `Artosyn in HS Mode`，可跳过此操作。

3. 在设备列表中选择 `Artosyn in HS Mode`。

4. 在驱动栏选择 `WinUSB`，例如 `WinUSB v6.1.7600.16385`。

5. 点击 `Install WCID Driver` 完成驱动安装。

6. 打开 Windows 设备管理器，在“通用串行总线设备”中确认能够识别到 `Artosyn in HS Mode`。

## 3\. 设备信息

`设备信息` 页面用于查看当前设备概况和管理对频。

主要信息包括：

- `工作模式`：显示当前链路工作模式，例如 1 对 1、1 对多等。

- `本机 MAC`：当前打开设备的 MAC 地址。

- `运行时间`：设备本次启动后的运行时长。

- `固件版本`：设备当前固件版本。

- `设备角色`：显示 AP 或 DEV 角色，角色由设备固件决定。

`对频管理` 表格按 slot 展示对端设备状态。常用操作：

- `查询`：读取当前 slot 设置的 MAC。

- `设置`：将输入框中的 MAC 写入当前 slot。

- `对频`：启动对频流程。点击对频后，按钮会切换为停止。

- `停止`：对频过程中点击可中止当前流程。

## 4\. 固件升级

`固件升级` 页面用于执行本机设备或对端设备固件升级。

![Image](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=MzE0OWE5ZmM0MTgyYjYxMmRlM2M3ZmU5ZGM5NjE0ZmFfYTk1NDdjZGMxOWIxMWMyZmVkOWMyZmQzMzAwYTg4YmFfSUQ6NzY1NDc2NzgxNTM2NzA2ODg3M18xNzgyMzc3MDA2OjE3ODI0NjM0MDZfVjM)

### 4\.1本地升级

操作步骤：

1. 在 `升级目标` 中选择 `本机升级` 或 `对端升级`（给对端设备升级固件）。

2. 点击 `选择文件`，选择符合要求的 `VT4*.img` 固件文件。

3. 确认文件名显示正确后，点击 `开始升级`。

4. 在确认弹窗中核对升级提示，再点击确认开始。

5. 等待进度条完成，期间不要断电、拔线或关闭工具。

重要注意事项：

- 对端升级要求当前设备已完成对频，并且存在可升级的远程 slot。

- 升级过程中请保持设备供电稳定。

- 升级成功后，工具会恢复配置文件出厂设置并重启设备。

### 4\.2 在线升级 

> 在线升级功能会根据当前设备固件版本解析 KT 型号（如 KT\-2458、KT\-1400 的 G/S、U/E 版本），从服务器检查可用固件；工具下载固件后会校验文件大小和校验和，并保存到本地固件缓存，因此用户不需要提前准备或手动保存固件文件。目前仅支持 KT 系列固件在线升级。
> 
> 

操作步骤：

1. 连接设备并确认顶部状态栏显示设备在线。

2. 在固件升级页面选择本机或对端升级目标；对端升级前需确认对端设备已连接。

3. 点击检查更新，工具会按当前固件版本匹配 KT 型号并向服务器查询可用版本。

4. 发现新版本后，在确认弹窗中核对目标版本并确认下载。

5. 下载完成后工具会校验固件大小和校验和，并把固件保存到本地缓存。

6. 点击开始升级，等待 OTA 进度完成；升级期间保持设备供电和连接稳定。

`链路监控` 页面用于查看链路指标和环境扫频。

### 5\.1链路信息

链路信息表格显示本机和对端的关键信息：

|**名称**|**说明**|
|---|---|
|频点（Freq）|当前设备链路发射信道的中心频率，用于确认本机与对端是否处在预期信道。|
|频宽（BandWidth）|链路占用的频谱宽度，影响抗干扰能力、可用速率和扫频图中的频宽范围。|
|发射功率（TxPower）|设备当前发射强度，功率越高覆盖能力通常越强，但也会增加干扰和功耗。|
|调制编码（MCS）|调制与编码方案索引，体现当前链路速率档位。MCS 越高，理论速率越高，对链路质量要求也越高。见附录 1。|
|接收信号强度（RSSI）|接收信号强度，用于判断信号是否足够稳定。|
|信噪比（SNR）|信噪比，用于衡量有效信号与噪声的差距，数值越高链路质量越好。|
|错误统计（Error）|展示链路收发过程中的异常计数，可用于判断是否存在干扰、弱信号。|
|吞吐率（Throughput）|显示实际吞吐与理论吞吐，便于判断链路性能是否达到当前 MCS 和频宽下的预期。|
|距离（Distance）|显示 AP 与 DEV 的测距结果；近距离误差较大，仅供参考。|

### 5\.2 环境扫频

环境扫频图用于观察 2G 和 5G 频段的干扰情况：

- 本机频点和对端频点以半透明矩形频宽框显示，矩形中心对应当前频点，宽度对应频宽范围，本机和对端用不同颜色区分。

- 本机扫频信号和对端扫频信号以不同颜色折线显示。

- 曲线越高，说明该频段环境越差，选频时应尽量避开。

### 5\.3 链路数据记录

链路数据记录：

- 点击 记录 开始记录当前链路指标。

- 记录过程中可点击 标记 为最近一次采样添加现场事件标记，工具会按顺序生成 Flag\_1、Flag\_2 等标记，便于回放时定位现场事件、异常波动或测试条件切换。

- 点击 `导出` 将记录保存为 CSV 文件。

导出的 CSV 可按时间、采样序号、标记和链路指标回放问题现场，方便定位异常发生前后的频点、信号质量、吞吐率和距离变化。

## 6\. 链路配置

链路配置 页面用于调整本机或对端射频参数。配置会即时生效，掉电后丢失；在运行期间优先级最高。

![Image](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=YzlkYWUzOWQ0NjEwZTMzMzdiMjQ2NjZkNjE3OTM3OGZfYWUzY2U1MzU3ODY1ZTE4MDA4NmIxYjQ0NDFiOWJjNTRfSUQ6NzY1NDc3NTc0NDc4Njk0Mjk0Ml8xNzgyMzc3MDA2OjE3ODI0NjM0MDZfVjM)

页面顶部的 `目标设备` 用于选择配置作用对象：

- `本机`：配置当前打开设备。

- `对端`：配置当前链路中的对端设备，需要链路正常，并且远程调用能力可用。

可配置项包括：

- `频段设置`：自动或手动选择 2G/5G。

> 注意：1\.4G 模块只能选择 2G 频段。
> 
> 

- `信道设置`：自动或手动选择信道。

- `频宽设置`：自动或手动选择 1\.25 MHz 到 40 MHz。

- `MCS`：自动或手动选择调制编码。

- `功率设置`：自动或手动选择发射功率。

- `带宽切换`：在默认和切换方向之间切换大带宽方向。

使用建议：

- 不确定现场频谱情况时，先在 `链路监控` 查看环境扫频，再调整频段和信道。

- 选择 `手动` 后，对应下拉框才会启用。

- 参数变化后请观察页面底部的配置状态，例如 `正在下发链路配置`、`链路配置已下发` 或失败提示。



## 7\. 持久配置

持久配置 页面用于管理设备 MiniDB 中的持久化参数。配置掉电后仍会保留，重启设备后生效；优先级低于链路配置。

![Image](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=ODIzMGU1MmIyYWE0NTBjNGU4YzllMjRiNjY0ZGFlNjJfNTcwYTBlMGJiZjM1NTAyMzVhNDY4NjEwNzc3NmY0NGNfSUQ6NzY1NTEzOTY4ODk3ODQ1MTY1NV8xNzgyMzc3MDA2OjE3ODI0NjM0MDZfVjM)

页面顶部的 `目标设备` 用于选择配置作用对象：

- `本机`：配置当前打开设备。

- `对端：配置当前链路中的对端设备，需要链路正常，并且远程调用能力可用。`

可配置项包括：

- `角色配置`：未设置、AP、DEV。

- `对端 MAC 地址`：写入对应 slot 的 MAC 地址。

- `频段设置`：未设置、自动、2G、5G。

- `功率设置`：未设置、固定值、范围。

- `频点列表设置`：输入频点列表，例如 `2400,2411,2422`。

写入规则：

- 分段按钮类配置在点击后写入。

- 文本或数字输入项通常在按 Enter 或输入框失焦后写入。

- 页面状态会显示 `配置已同步`、`配置已修改`、`正在写入配置` 或 `等待重启生效`。

底部操作：

- `清除配置`：清除设备 MiniDB 持久配置，执行前会弹出确认。

- `重启生效`：重启设备使持久配置生效，执行后设备会短暂断开连接。



## 8\. 配置文件

配置文件 页面用于读取、编辑、导入、导出设备 JSON 配置。配置文件提供默认参数。若链路配置和持久配置都未设置，设备会使用配置文件中的配置；该配置优先级最低。

页面底部用于选择配置作用对象：

- `本机`：配置当前打开设备。

- `对端`：配置当前链路中的对端设备，需要链路和远程调用能力可用。

常用操作：

- `读取`：从设备读取当前 JSON 配置并显示在编辑区。

- `保存`：将编辑区中的 JSON 写入设备，写入前会校验 JSON 格式并弹出确认。

- `导入`：从本地文件导入 JSON 到编辑区。

- `导出`：将编辑区中的 JSON 保存到本地文件。

- `恢复出厂`：将设备配置恢复为出厂默认值。

- 搜索框：在 JSON 内容中查找配置项，可使用上一个/下一个按钮跳转。

注意：

- 点击 `保存` 会覆盖设备当前配置，请先确认 JSON 内容正确。

- `恢复出厂` 属于高风险操作，会覆盖当前设备配置，执行前请先导出备份。

- 导入文件只改变编辑区内容；写入设备需要再点击 `保存` 并确认。

## 10\. 运行状态与日志

窗口底部状态栏用于判断软件当前状态：

- 左侧圆点和文字表示 `l4_daemon` 连接状态。

- 中间 `运行日志` 显示最近一次操作结果，例如设备打开、配置读取、持久配置已读取等。

- 右侧显示软件版本和 daemon 地址。

如果操作失败，请优先查看底部运行日志，再检查设备连接、daemon 状态、对频状态和输入参数。

## 11\. 常见问题

### 设备下拉框没有设备

请检查设备是否正确连接、供电是否稳定、驱动是否正常，并确认底部状态栏不是 `l4_daemon 未连接`。

### 对端升级不可用

远程升级需要对端设备已对频并可通过当前链路访问。请先在 `设备信息` 页确认 slot 连接状态，再尝试远程升级。

### 链路配置下发失败

请确认当前设备在线；如果配置目标是对端，请确认对端已连接。必要时切回 `链路监控` 查看链路是否就绪。

### JSON 保存失败

请先确认编辑区内容是合法 JSON。保存前建议先导出备份，避免覆盖有效配置。

### 持久配置写入后没有立即生效
