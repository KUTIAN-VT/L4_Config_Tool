# L4 配置工具软件使用手册

本文面向 L4 配置工具的日常使用人员，说明软件启动、设备连接、状态查看、对频管理、固件升级、链路监控、链路配置、持久配置和默认配置维护等常用操作。

界面截图以实际运行窗口为准。设备在线状态、固件版本、MAC 地址、链路指标和运行日志会随现场设备变化。

## 1. 运行前准备

### 1.1 检查软件发布包

使用前请确认发布包已完整解压，并从解压后的目录中运行软件。不要只复制单个 `L4_Config_Tool_Web.exe` 到其他目录运行。

发布目录中至少应包含以下内容：

- `L4_Config_Tool_Web.exe`：配置工具主程序。
- `l4_daemon.exe`：设备通信守护进程。
- `l4_ota_upgrade.exe`：OTA 升级工具。
- `ar8030_client.dll`、`pthread.dll`、Qt 运行库、`web/` 目录和 `qt.conf` 等运行依赖。

### 1.2 连接设备

请在启动软件前完成以下检查：

- L4 设备已通过 USB连接到 Windows 电脑。
- 设备供电稳定，升级或写配置期间不要断电。

### 1.3 安装 WinUSB 驱动

首次使用或更换电脑后，需要确认 Windows 能识别 L4 设备。

1. 运行解压目录中的 `zadig.exe`。
2. 点击菜单栏 `Options`，选择 `List All Devices`。如果界面已经显示 `Artosyn in HS Mode`，可跳过此操作。
3. 在设备列表中选择 `Artosyn in HS Mode`。
4. 在驱动栏选择 `WinUSB`，例如 `WinUSB v6.1.7600.16385`。
5. 点击 `Install WCID Driver` 完成驱动安装。
6. 打开 Windows 设备管理器，在“通用串行总线设备”中确认能够识别到 `Artosyn in HS Mode`。

## 2. 启动与设备连接

1. 双击运行 `L4_Config_Tool_Web.exe`。
2. 等待底部状态栏显示 `l4_daemon 已连接`。
3. 在右上角设备下拉框中选择需要操作的设备。
4. 设备打开后，左侧功能区会显示 `设备信息`、`固件升级`、`链路监控`、`链路配置`、`持久配置`、`默认配置` 和 `设置`。

如果设备列表为空，请按以下顺序排查：

1. 确认设备已上电并连接到电脑。
2. 确认设备管理器中能看到 `Artosyn in HS Mode`。
3. 确认发布目录中的 `l4_daemon.exe` 未被杀毒软件拦截。
4. 查看底部状态栏和运行日志，确认是否存在 daemon 未连接、设备打开失败或权限错误。

## 3. 基本概念

### 3.1 本机与对端

工具中的 `本机` 指当前电脑直接打开的 L4 设备，`对端` 指当前链路另一端的设备。

选择 `对端` 前，请先确认链路已经建立，并且对端在 `设备信息` 页中显示为已连接。仅处于对频流程中并不等于对端已经可以被配置或升级。

### 3.2 三类配置的生效关系

工具中有三类配置入口，优先级不同：

| 配置入口 | 页面名称 | 生效方式 | 掉电后是否保留 | 优先级 |
|---|---|---|---|---|
| 运行时链路配置 | `链路配置` | 下发后即时生效 | 不保留 | 最高 |
| MiniDB 持久配置 | `持久配置` | 写入后需要重启生效 | 保留 | 中 |
| JSON 默认配置 | `默认配置` | 设备读取默认参数时生效 | 保留 | 最低 |

如果同一参数在多个入口中都被设置，以优先级更高的配置为准。现场临时调参建议使用 `链路配置`；需要重启后仍保留的参数建议写入 `持久配置`；需要维护完整 JSON 默认参数时使用 `默认配置`。

## 4. 设备信息

`设备信息` 页面用于查看当前设备概况，并管理对频关系。

页面顶部显示以下信息：

- `工作模式`：显示当前链路工作模式，例如 1V1、1VN 等。
- `本机 MAC`：当前打开设备的 MAC 地址。
- `运行时间`：设备本次启动后的运行时长。
- `固件版本`：当前设备固件版本。
- `设备角色`：显示 AP 或 DEV，角色由设备固件和配置决定。

`对频管理` 表格按 slot 展示对端状态。常用操作如下：

- `查询`：读取当前 slot 已保存的对端 MAC。
- `设置`：将输入框中的 MAC 写入当前 slot。
- `对频`：启动对频流程。
- `停止`：对频过程中停止当前流程。

对端升级、对端链路配置、对端持久配置和对端默认配置都依赖可用的远程链路。操作前请先在此页面确认对应 slot 已连接。

## 5. 固件升级

`固件升级` 页面用于升级本机设备或对端设备固件。

![固件升级页面，包含升级目标、固件文件、进度和检查更新按钮](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=MzE0OWE5ZmM0MTgyYjYxMmRlM2M3ZmU5ZGM5NjE0ZmFfYTk1NDdjZGMxOWIxMWMyZmVkOWMyZmQzMzAwYTg4YmFfSUQ6NzY1NDc2NzgxNTM2NzA2ODg3M18xNzgyMzc3MDA2OjE3ODI0NjM0MDZfVjM)

### 5.1 使用本地固件文件升级

当已经拿到本地固件文件时，按以下步骤操作：

1. 在 `升级目标` 中选择 `本机升级` 或 `对端升级`。
2. 点击 `选择文件`，选择文件名以 `VT4` 开头且扩展名为 `.img` 的固件文件，例如 `VT4*.img`。
3. 确认文件名显示正确后，点击 `开始升级`。
4. 在确认弹窗中核对升级提示，再点击 `开始升级`。
5. 等待进度条完成。升级期间不要断电、拔线或关闭工具。

注意事项：

- `本机升级` 升级当前电脑直接打开的设备。
- `对端升级` 升级当前链路中的对端设备，要求对端已连接。
- 升级成功后，工具会自动执行配置文件恢复出厂并重启设备。
- 如果升级失败，请先查看底部运行日志，再检查固件文件名、设备连接、对端连接状态和供电情况。

### 5.2 在线升级

在线升级用于从服务器检查并下载可用固件。工具会根据当前设备固件版本解析 KT 型号，目前支持 KT 系列固件，例如 KT-2458、KT-1400 的 G/S、U/E 版本。

操作步骤：

1. 连接设备，并确认顶部设备区域和底部状态栏显示设备在线。
2. 在 `升级目标` 中选择 `本机升级` 或 `对端升级`。选择 `对端升级` 前，请确认对端已连接。
3. 点击 `检查更新`。工具会根据当前目标的固件版本匹配设备类型，并向服务器查询可用版本。
4. 如果发现新版本，确认弹窗会显示目标版本。点击 `下载并升级` 后，工具会下载固件并在下载完成后自动开始 OTA。
5. 等待进度条完成。升级期间保持设备供电和连接稳定。

在线升级说明：

- 下载固件后，工具会校验文件大小和校验和。
- 下载到的固件会保存到本地缓存，用户不需要手动保存固件文件。
- 如果提示当前固件版本无法匹配 ThingsBoard 设备类型，请确认设备固件版本号是否符合 KT 型号命名规则。

### 6.1 链路信息

链路信息表格会分别显示 `本机` 和 `对端` 的关键指标：

| 名称 | 说明 |
|---|---|
| 频点 | 当前链路发射信道的中心频率，用于确认设备是否工作在预期频段。 |
| 频宽 | 链路占用的频谱宽度，会影响抗干扰能力和可用速率。 |
| 发射功率 | 当前发射强度。功率越高通常覆盖能力越强，但也可能增加干扰和功耗。 |
| 调制编码 | MCS 档位。MCS 越高，理论速率越高，对链路质量要求也越高。对应关系见附录 1。 |
| 信号强度 | RSSI，用于判断接收信号是否足够稳定。 |
| 信噪比 | SNR，用于衡量有效信号与噪声的差距，数值越高通常链路质量越好。 |
| 错误统计 | 链路收发过程中的异常计数，可用于判断干扰、弱信号或传输异常。 |
| 吞吐率 | 显示实际吞吐和理论吞吐，便于判断链路性能是否符合当前 MCS 和频宽预期。 |
| 距离 | AP 与 DEV 的测距结果。近距离误差可能较大，仅供参考。 |

### 6.2 环境扫频

环境扫频图用于观察 2G 和 5G 频段的干扰情况：

- 横轴是频点，纵轴是信号干扰强度。
- 图例从 `弱干扰` 到 `强干扰` 表示干扰强弱。
- 本机频点和对端频点以半透明频宽框显示，矩形中心对应当前频点，宽度对应当前频宽范围。
- 本机扫频信号和对端扫频信号以不同颜色折线显示。
- 曲线越高，说明该频段环境越差。手动选频时应尽量避开干扰较强的频点。

### 6.3 链路数据记录

链路数据记录用于在现场测试时保存链路变化过程。

1. 点击 `记录` 开始记录当前链路指标。
2. 记录过程中可点击 `标记`，为最近一次采样添加现场事件标记。工具会按顺序生成 `Flag_1`、`Flag_2` 等标记。
3. 点击 `停止` 结束当前记录。
4. 点击 `导出`，将记录保存为 CSV 文件。

导出的 CSV 可按时间、采样序号、标记和链路指标回放问题现场，方便定位异常发生前后的频点、信号质量、吞吐率和距离变化。

## 7. 链路配置

`链路配置` 页面用于调整本机或对端的运行时射频参数。配置下发后即时生效，但掉电后会丢失；在运行期间优先级最高。

![链路配置页面，包含目标设备、频段、信道、频宽、MCS、功率和带宽切换](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=YzlkYWUzOWQ0NjEwZTMzMzdiMjQ2NjZkNjE3OTM3OGZfYWUzY2U1MzU3ODY1ZTE4MDA4NmIxYjQ0NDFiOWJjNTRfSUQ6NzY1NDc3NTc0NDc4Njk0Mjk0Ml8xNzgyMzc3MDA2OjE3ODI0NjM0MDZfVjM)

页面顶部的 `目标设备` 用于选择配置作用对象：

- `本机`：配置当前打开的设备。
- `对端`：配置当前链路中的对端设备。对端操作需要链路正常，并且远程调用能力可用。

可配置项包括：

- `频段设置`：选择自动或手动。手动模式可选择 2G 或 5G。1.4G 模块只能选择 2G。
- `信道设置`：选择自动或手动。手动模式下从下拉框选择信道。
- `频宽设置`：选择自动或手动。手动模式支持 1.25 MHz、2.5 MHz、5 MHz、10 MHz、20 MHz 和 40 MHz。
- `MCS`：选择自动或手动。可选值会根据设备角色和默认配置动态调整。
- `功率设置`：选择自动或手动。手动模式下从可用功率范围中选择发射功率。
- `带宽切换`：在默认方向和切换方向之间切换大带宽方向。

使用建议：

- 不确定现场频谱情况时，先在 `链路监控` 查看环境扫频，再调整频段和信道。
- 选择 `手动` 后，对应下拉框才会启用。
- 每次参数变化后，请观察页面底部状态，例如 `正在下发链路配置`、`链路配置已下发` 或 `链路配置下发失败`。
- 如果对端配置失败，请先回到 `设备信息` 确认对端已连接。

## 8. 持久配置

`持久配置` 页面用于管理设备 MiniDB 中的持久化参数。此类配置掉电后仍会保留，通常需要重启设备后完全生效；优先级低于 `链路配置`。

页面顶部的 `目标设备` 用于选择配置作用对象：

- `本机`：配置当前打开的设备。
- `对端`：配置当前链路中的对端设备。对端操作需要链路正常，并且远程调用能力可用。

可配置项包括：

- `角色配置`：未设置、AP、DEV。
- `对端 Mac地址`：写入对应 slot 的对端 MAC 地址。
- `频段设置`：未设置、自动、2G、5G。

- `重启生效`：重启目标设备，使持久配置生效。执行后设备会短暂断开连接，请等待设备重新上线。

## 9. 默认配置（配置文件）

`默认配置` 页面用于读取、编辑、导入、导出设备 JSON 配置。该 JSON 是设备默认参数来源；当运行时链路配置和 MiniDB 持久配置都未覆盖相关参数时，设备会使用默认配置中的参数。

![默认配置页面，显示 JSON 配置编辑区和读取、保存、导入、导出、恢复出厂按钮](https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=ZWMxMTg5ZDNjMjNjZjk0NDE3NWIxOTJkYTE2NjU0YTdfNDBkOGQwNTE3MjlhZGZkYWJkYjE4YzgxYWJjMDJjNWFfSUQ6NzY1NTEzOTc3NDczNDg5NjEwN18xNzgyMzc3MDA2OjE3ODI0NjM0MDZfVjM)

页面底部用于选择配置作用对象：

- `本机`：读写当前打开设备的 JSON 默认配置。
- `对端`：读写当前链路中对端设备的 JSON 默认配置。对端操作需要链路和远程调用能力可用。

常用操作：

- `读取`：从目标设备读取当前 JSON 配置，并显示在编辑区。
- `保存`：将编辑区中的 JSON 写入目标设备。写入前工具会校验 JSON 格式并弹出确认。
- `导入`：从本地文件导入 JSON 到编辑区。导入不会立即写入设备。
- `导出`：将编辑区中的 JSON 保存到本地文件。
- `恢复出厂`：将目标设备的 JSON 默认配置恢复为出厂默认值。
- 搜索框：在 JSON 内容中查找配置项，可使用上一个/下一个按钮跳转。

安全建议：

- 修改前先点击 `读取`，确认编辑区内容来自当前目标设备。
- 保存前先点击 `导出` 备份当前 JSON。
- `保存` 会覆盖目标设备当前配置。
- `恢复出厂` 属于高风险操作，会覆盖当前设备配置，执行前请先导出备份。
- JSON 格式不合法时，工具会阻止保存并提示错误。

### 9.1 功率配置

固定功率：将 `manu_init` 改为目标功率，并将 `pwr_auto` 改为 `manu`。

## 12. 常见问题

### 设备下拉框没有设备

请检查设备是否正确连接、供电是否稳定、驱动是否正常，并确认底部状态栏不是 `l4_daemon 未连接`。

### 底部状态栏显示 l4_daemon 未连接

请确认发布目录完整，`l4_daemon.exe` 与主程序在同一目录；同时检查杀毒软件或系统权限是否阻止守护进程启动。

### 对端升级不可用

对端升级需要对端设备已经通过当前链路连接。请先在 `设备信息` 页面确认 slot 连接状态，再尝试对端升级。

### 在线升级检查失败

请先确认设备在线、网络可访问服务器，并查看底部运行日志。如果提示当前版本异常，通常表示当前固件版本号无法匹配在线升级支持的 KT 设备类型。

### 链路配置下发失败

请确认当前设备在线。如果配置目标是 `对端`，请确认对端已连接。必要时切回 `链路监控` 查看链路是否就绪。

### JSON 保存失败

请先确认编辑区内容是合法 JSON。保存前建议先导出备份，避免覆盖有效配置。

### 持久配置写入后没有立即生效

部分持久化参数需要设备重启后生效。请根据页面状态提示点击 `重启生效`，并等待设备重新上线。
